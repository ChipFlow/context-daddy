# Context Daddy Promotional Plan

## Goals

1. **Awareness**: Get context-daddy known in the Claude Code community
2. **Adoption**: Drive installs and active usage
3. **Feedback**: Learn what resonates and iterate

## Target Audience

- **Primary**: Claude Code power users frustrated with context loss
- **Secondary**: Developers exploring AI-assisted coding
- **Tertiary**: Teams wanting to capture tribal knowledge

## Key Messages

1. "Your codebase's context needs a responsible adult"
2. Fast symbol search (10-100x faster than grep)
3. Living narratives that survive compaction
4. Tribal knowledge that doesn't get lost

## Dynamic Adjustment Framework

### Signal → Response Matrix

| Signal | Interpretation | Response |
|--------|---------------|----------|
| High social engagement, low stars | Content resonates but not converting | Add clearer CTA, simplify install |
| High stars, low issues | Users like it but not engaging deeply | Add prompts for feedback, improve docs |
| Lots of bug reports | Tool has rough edges | Pause promo, fix quality |
| Feature requests clustering | Unmet need identified | Prioritize that feature, announce it |
| HN front page | Moment of high visibility | All hands on engagement, fast issue response |
| Crickets everywhere | Message not landing | Pivot messaging, try different channels |

### A/B Testing Opportunities

1. **Taglines**: Test different value props
   - "Your codebase's context needs a responsible adult"
   - "Stop losing context. Start understanding code."
   - "10x faster code navigation for Claude"

2. **Visuals**: Test logo variations, demo formats

3. **Channels**: Track which drives most quality traffic
